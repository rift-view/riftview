## Background: What's Wrong

**`build:win` re-runs typecheck:**
- `build:win` = `npm run build && electron-builder --win`
- `npm run build` = `npm run typecheck && electron-vite build`
- So Windows builds re-run typecheck even though the `ci` job already validated it. Fix: align with mac/linux pattern.

**Release job never calls electron-builder:**
- Current release job runs `npm run build` (= typecheck + vite) — no packaging step
- `dist/**` contains only Vite output, not `.dmg` / `.exe` / `.AppImage` installers
- Job runs on `ubuntu-latest` only — macOS `.dmg` is impossible on Ubuntu

**Publish strategy:**
- `electron-builder.yml` has `publish: provider: github` already configured
- Pass `GH_TOKEN` to the build step; call `electron-builder --publish always` directly
- No need for a separate `softprops/action-gh-release@v2` step — electron-builder handles it

## Task 2: Fix CI Release Pipeline

**Files:**
- Modify: `.github/workflows/ci.yml`

- [ ] **Step 1: Read current ci.yml**

```bash
cat .github/workflows/ci.yml
```

- [ ] **Step 2: Replace the `release:` job**

Delete the entire `release:` block and replace with:

```yaml
  release:
    needs: ci
    if: startsWith(github.ref, 'refs/tags/v')
    strategy:
      matrix:
        include:
          - os: macos-latest
            platform: mac
          - os: windows-latest
            platform: win
          - os: ubuntu-latest
            platform: linux
    runs-on: ${{ matrix.os }}
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: npm
          cache-dependency-path: riftview/package-lock.json
      - name: Install dependencies
        run: npm ci
        working-directory: riftview
      - name: Build and publish
        run: npx electron-vite build && npx electron-builder --${{ matrix.platform }} --publish always
        working-directory: riftview
        env:
          GH_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

**Why `npx electron-vite build && npx electron-builder --publish always` instead of `npm run build:mac`:**
- Avoids npm script indirection
- `--publish always` tells electron-builder to upload artifacts to the GitHub Release using `GH_TOKEN`
- electron-builder reads the publish config from `electron-builder.yml` automatically
- No separate upload step needed — electron-builder handles it
